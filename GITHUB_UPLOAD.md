# Инструкция по загрузке проекта в GitHub

## Вариант 1: Через командную строку (рекомендуется)

### Шаг 1: Инициализируйте Git репозиторий

Откройте терминал в папке проекта и выполните:

```bash
git init
```

### Шаг 2: Добавьте все файлы

```bash
git add .
```

### Шаг 3: Создайте первый коммит

```bash
git commit -m "Initial commit - School Social Network"
```

### Шаг 4: Создайте репозиторий на GitHub

1. Зайдите на https://github.com
2. Нажмите кнопку **"New"** или **"+"** → **"New repository"**
3. Назовите репозиторий (например: `school-social-network`)
4. **ВАЖНО:** НЕ добавляйте README, .gitignore или лицензию (они уже есть)
5. Нажмите **"Create repository"**

### Шаг 5: Подключите локальный репозиторий к GitHub

После создания репозитория GitHub покажет вам команды. Выполните:

```bash
git branch -M main
git remote add origin https://github.com/ВАШ-USERNAME/ВАШ-РЕПОЗИТОРИЙ.git
git push -u origin main
```

**Замените:**
- `ВАШ-USERNAME` на ваш GitHub username
- `ВАШ-РЕПОЗИТОРИЙ` на название вашего репозитория

### Шаг 6: Введите логин и пароль

GitHub попросит вас авторизоваться. Используйте:
- **Username:** ваш GitHub username
- **Password:** Personal Access Token (не обычный пароль!)

**Как создать Personal Access Token:**
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Generate new token (classic)
3. Выберите срок действия и права: `repo` (все права репозитория)
4. Скопируйте токен и используйте его как пароль

## Вариант 2: Через GitHub Desktop (проще)

### Шаг 1: Установите GitHub Desktop

Скачайте с https://desktop.github.com

### Шаг 2: Откройте проект

1. File → Add Local Repository
2. Выберите папку `school-social-network`
3. Нажмите "Add repository"

### Шаг 3: Создайте репозиторий на GitHub

1. В GitHub Desktop нажмите "Publish repository"
2. Введите название репозитория
3. Снимите галочку "Keep this code private" (если хотите публичный репозиторий)
4. Нажмите "Publish Repository"

Готово! Код загружен на GitHub.

## Проверка

После загрузки зайдите на GitHub и проверьте:
- ✅ Все папки видны (app, components, lib и т.д.)
- ✅ Все файлы загружены
- ✅ README.md отображается

---

## Если папки не видны

### Проблема: Папки пустые или не видны

**Решение:**
1. Проверьте `.gitignore` - возможно, папки игнорируются
2. Убедитесь, что выполнили `git add .` (точка в конце важна!)
3. Проверьте, что файлы не в `node_modules` или других игнорируемых папках

### Проблема: Ошибка "fatal: not a git repository"

**Решение:**
Выполните `git init` в папке проекта

### Проблема: Ошибка при push

**Решение:**
- Проверьте, что репозиторий создан на GitHub
- Проверьте правильность URL репозитория
- Используйте Personal Access Token вместо пароля

---

## Что НЕ нужно загружать

Следующие папки/файлы автоматически игнорируются (благодаря `.gitignore`):
- `node_modules/` - зависимости (устанавливаются через `npm install`)
- `.next/` - скомпилированные файлы
- `data/` - база данных (создается автоматически)
- `.env.local` - секретные ключи

Это нормально! Эти файлы не нужны в репозитории.
